No, the play buttons don’t appear on Tableau Server either. It’s annoying, but given the architecture of Tableau Server/Public, it’s fairly reasonable. The data lives on the server. If you are going to animate a viz, the data has to get sent from the server for each “frame”. Maybe that’s possible for a view with only a few data points.

But what if we tried it on the Gapminder view above? There’s about 100 data points on there, not including the trails, if you’ve got them on. There’s no way Tableau Server/Public could get the data down to the browser quickly enough.

How do other online viz applications solve this? Well, the “real” Gapminder seems to load the data in the browser.
