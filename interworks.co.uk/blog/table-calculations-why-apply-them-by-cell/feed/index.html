from a blank worksheet,
1. place Product Type on the Columns shelf
2. place Market on the Rows shelf
(notice there are only 15 marks for 4 columns and 4 rows)
3. place Sales on the Text shelf
4. make the Sales pill a Running Total Quick Table Calc, this defaults to a Compute Using of Table (Across)
(notice there are now 16 marks)

When Tableau adds marks for dimension value combinations that do not exist in the data source, this is called Data Densification. The example here is just one type, there are other types and other ways of dealing with them.

How do you turn turn Data Densification off in this case?
or in other words,
How do you make the extra mark go away without impacting the computation?

Answer:
1. set the Compute Using to Product Type
1. place another pill for Product Type on the Detail shelf
2. from the context menu on the Product Type pill on the Columns shelf, change it from a Dimension to an Attribute.

Important here is awareness of the 4 Pill Types, and the impact of using either a Discrete Dimension to a Discrete Measure on the Columns/Rows shelves at the same time with a table calculation and its Compute Using setting.

Here is another way to think of these options:

Table Down: Address on Dimension pills on Rows shelf, Partitioning on all other Dimension pills

Table Across: Address on Dimension pills on Columns shelf, Partitioning on all other Dimension pills

Cell: Address on Dimension pills on Marks card, Partitioning on all other Dimension pills

Also, TOTAL() is an exception when using Table/Pane options, when using TOTAL() and Table or Pane compute usings, the Dimension pills on the Marks card will also be used for Addressing.
