]]> By: Kurt http://interworks.co.uk/blog/menu-navigation-within-a-tableau-workbook/#comment-43382 Mon, 13 Jan 2014 16:18:05 +0000 http://www.interworks.co.uk/?p=3249#comment-43382 Re: Sudharshan’s comment:

“When I Click on Map, it navigates to Map Tab but when i Click Back and come to the Main screen, I can’t click Map again as it is already selected. How do you automatically unselect Map when clicking the Back button in Maps tab ?”

I would love to see a solution for this. It’s a problem with any navigation technique that uses worksheets for the navigation element and filter actions. If you keep the navigation elements in every dashboard (like a set of links), the problem is worse because the user will be on View B but the navigation button for his previous view, View A, will still appear “selected.”

You can see a good example of this here:

http://public.tableausoftware.com/views/DiabetesandTBintheAmericas2008TPteasers/IncidenceofTB?:embed=y#1

1. At the bottom of the viz, click twice on the first filmstrip thumbnail (TB inc rate) to ensure you are on that tab.
2. Now click on the second filmstrip thumbnail, DM Prevalence.
3. Now click on the first filmstrip thumbnail , TB inc rate. You are now back on the first tab (TB inc rate), but there is a black box around the DM Prevalence filmstrip thumbnail because it has not been completely deselected. It is confusing to the user.

If this doesn’t work the first time, experiment with going back and forth among various thumbnails and eventually you’ll see the issue.

]]> By: Aus - Blog Author http://interworks.co.uk/blog/menu-navigation-within-a-tableau-workbook/#comment-438 Fri, 01 Feb 2013 09:29:45 +0000 http://www.interworks.co.uk/?p=3249#comment-438 Hi Daniel,

Thanks for your question.

Navigating around a workbook with this method requires the tabs to be published. The way to achieve this without having to show each tab along the top is to navigate to the exact (and full) URL.

The best way to do this is to publish the workbook to your sever (or public). Then from within server navigate to each of the pages (tabs) directly and make a note of each url. Just flicking through the tabs won’t change the url. Then add those url’s into the navigation of your workbook and republish. In the past I’ve make a new data source like a csv with the url’s in it for ease of maintenance.

Hope that helps.
